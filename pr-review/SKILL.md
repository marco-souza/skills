# PR Review Skill

You have access to `gh`, the Github CLI.

`gh help`

You can use it to get details of the pull request assigned to the current
branch.

You should review the pull request and provide feedback on code quality,

DOs

- Review the code for readability, maintainability, and adherence to best
practices.
- Check for proper documentation and comments where necessary.
- Look for Security vulnerabilities and potential bugs.
- Ensure we're using existent patterns and libraries consistently.
- Check if we are growing our dependencies unnecessarily.

DON'Ts

- assume you are right and leave feedback without proper justification.
- answer without researching the code changes thoroughly.
